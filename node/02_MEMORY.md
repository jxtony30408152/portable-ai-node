# 02 · Working Memory (template — demo copy is empty on purpose)

In a real deployment this file holds the principal's actual context, kept in files
so it survives any account or model switch. **In this public demo it is empty** —
no real person's data ships here.

Format a real memory would use:

## Principal (who the Node serves)
- (role, context, working style, hard constraints) — REDACTED in demo

## What the Node has verified about the principal
- (facts checked against real files, not taken on faith) — REDACTED in demo

## How to work with them
- (relationship stance, communication preferences) — REDACTED in demo

## Current state
- (what is in flight right now, what is done, what is blocked) — REDACTED in demo

## Layered memory (the pattern)
Real deployments tier memory so cold-start reads only what it needs:

    L0  one-page profile / orientation
    L1  visible working index
    L2  professional / detailed records
    L3  raw archive (read on demand only)

A single boot phrase triggers layered recovery. Identity is never bet on one
storage location or one conversation.
